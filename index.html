Wayfinder - Indoor Navigation Mobile App

This indoor navigation project was undertaken with the goal of developing a functioning prototype for navigating indoor environments, such as campuses. While the objective was to create a deployable product, the project served primarily as a learning tool, allowing the team to delve into the complexities of app development and interactive design.

Although the project aimed at deployment, several challenges prevented full realization, including technological limitations and time constraints. The experience provided crucial insights into the iterative development process, emphasizing the importance of flexibility and continuous learning in the face of unexpected hurdles.

  • Figma: Used for UI/UX design, creating detailed wireframes and interactive prototypes.
  • Flutter: Applied for building the mobile interface, ensuring cross-platform functionality.
  • Laravel & MySQL: Integrated as part of the backend framework to manage data handling and storage solutions.
  • Version Control with Git: Essential for managing code changes and facilitating effective team collaboration.
  • Project Management with Trello: Enabled tracking of tasks, deadlines, and project milestones.
  • Collaborative Learning: The project emphasized the value of teamwork and collective problem-solving within a digital project setting.

Portfolio-REST API

The project is a REST API back-end portfolio designed to showcase my skills in API development. It includes core functionalities such as user management, authentication, CRUD operations, etc. Instead of integrating a front-end, I used Postman for testing and validating the API endpoints. The project was managed using GitHub's Kanban board, with proper testing using Pest and Xdebug for code coverage.

  • Postman: Utilized for testing and validating API endpoints during development.
  • SQLite: Lightweight database used for development and storage, ensuring simplicity and efficiency.
  • Scribe: Used to generate comprehensive and easy-to-read API documentation.
  • Pest: Implemented for unit and feature testing to ensure code reliability.
  • Xdebug: Integrated for debugging and code coverage analysis.
  • GitHub Kanban Board: Managed the project workflow effectively with task tracking and progress visualization.
  • Git: Version control system for managing code changes and collaboration.
